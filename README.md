# **ภาจกิจที่ 1 กำหนดแนวคิด (Conceptual Framework)**

# **1. คำถามวิจัย**

งานวิจัยนี้มีวัตถุประสงค์เพื่อวิเคราะห์และสร้างแผนที่ความเสี่ยงน้ำท่วมในจังหวัดสงขลา โดยตอบคำถามว่าพื้นที่ใดมีความเสี่ยงต่อการเกิดน้ำท่วมมากหรือน้อยจากปัจจัยเชิงพื้นที่ที่เกี่ยวข้อง

**2. ปัจจัยที่ส่งผล (อิงวิทยาศาสตร์)**

ปัจจัยที่ส่งผลต่อการเกิดน้ำท่วม ได้แก่ ความสูงของพื้นที่ (Elevation) และความลาดชัน (Slope) ซึ่งมีผลต่อการไหลและการสะสมของน้ำ ปริมาณน้ำฝน (Rainfall) ซึ่งเป็นแหล่งกำเนิดน้ำ ระยะห่างจากแหล่งน้ำ (Distance to Water) ซึ่งสัมพันธ์กับโอกาสการเอ่อล้น และการใช้ประโยชน์ที่ดิน (Land Cover) ซึ่งมีผลต่อการซึมน้ำและการไหลบ่าของน้ำ โดยอ้างอิงตามหลักการทางอุทกวิทยา (Hydrology)

**3. การกำหนดน้ำหนัก**

การกำหนดน้ำหนักของแต่ละปัจจัยอ้างอิงจากแนวคิด Expert Knowledge โดยพิจารณาความสำคัญของปัจจัยตามหลักอุทกวิทยา เช่น ความสูงของพื้นที่และปริมาณน้ำฝนมีอิทธิพลสูงต่อการเกิดน้ำท่วม จึงให้ค่าน้ำหนักมากกว่าปัจจัยอื่น ทั้งนี้มีการปรับค่าน้ำหนักให้เหมาะสมกับบริบทของพื้นที่ศึกษา แทนการใช้ค่ามาตรฐานจากแหล่งอ้างอิงโดยตรง

**4. การตรวจสอบ (Validation)**

การตรวจสอบความถูกต้องของโมเดลใช้ข้อมูลจาก JRC Global Surface Water (Occurrence) เป็นตัวแทนของพื้นที่ที่มีโอกาสเกิดน้ำท่วมในระยะยาว โดยนำมาเปรียบเทียบกับผลลัพธ์ของโมเดลผ่านการคำนวณค่า Accuracy และการวิเคราะห์เชิงพื้นที่ เพื่อประเมินประสิทธิภาพของโมเดล

# **ภารกิจที่ 2: สร้างและรัน Spatial Model**
**1. การกำหนด Weight และ Combine**

ในการสร้างแบบจำลองความเสี่ยงน้ำท่วม ได้มีการกำหนดน้ำหนักของแต่ละปัจจัยโดยอ้างอิงจากแนวคิด Expert Knowledge ทางด้านอุทกวิทยา โดยให้ความสำคัญกับความสูงของพื้นที่ (Elevation) และปริมาณน้ำฝน (Rainfall) มากที่สุด รองลงมาคือระยะห่างจากแหล่งน้ำ (Distance to Water) ความลาดชัน (Slope) และการใช้ที่ดิน (Land Cover) จากนั้นนำปัจจัยทั้งหมดมารวมกันด้วยวิธี Weighted Linear Combination ในรูปแบบสมการเชิงเส้น เพื่อคำนวณค่าดัชนีความเสี่ยงน้ำท่วม (Flood Risk Index)

**2. การ Normalize แต่ละ Factor (วิธีและเหตุผล)**

ปัจจัยแต่ละตัวถูกปรับค่าให้อยู่ในช่วง 0–1 ด้วยวิธี Min-Max Normalization เพื่อให้สามารถเปรียบเทียบและรวมกันได้อย่างเหมาะสม โดยมีการปรับทิศทางของค่าบางปัจจัย เช่น Elevation และ Slope ให้มีความสัมพันธ์เชิงผกผันกับความเสี่ยงน้ำท่วม (ค่าต่ำ = เสี่ยงสูง) ในขณะที่ Rainfall และ Land Cover ใช้ค่าตรง (ค่าสูง = เสี่ยงสูง) ซึ่งสอดคล้องกับหลักการทางอุทกวิทยา

**3. การแบ่งระดับความเสี่ยง (Thresholds) — มาจากไหน**

การแบ่งระดับความเสี่ยงของน้ำท่วมถูกกำหนดโดยใช้ค่า threshold จากการกระจายตัวของข้อมูล (เช่น percentile หรือค่าช่วงของ Flood Risk Index) เพื่อจำแนกพื้นที่ออกเป็นระดับความเสี่ยงต่ำ ปานกลาง และสูง โดยอ้างอิงจากลักษณะการกระจายของข้อมูลจริงในพื้นที่ศึกษาแทนการกำหนดค่าคงที่

**4. แผนที่ผลลัพธ์**

ผลลัพธ์ของโมเดลถูกแสดงในรูปแบบแผนที่เชิงพื้นที่ที่สามารถตีความได้ง่าย โดยใช้การไล่ระดับสีเพื่อแสดงความแตกต่างของความเสี่ยงน้ำท่วม ทำให้สามารถระบุพื้นที่เสี่ยงสูงและต่ำได้อย่างชัดเจนในระดับจังหวัด

# **ภารกิจที่ 3: Sensitivity Analysis**
**1. การเปลี่ยนน้ำหนัก ±20%**

ได้ทำการทดสอบความไวของโมเดลโดยปรับค่าน้ำหนักของปัจจัยที่สำคัญที่สุด (Elevation) เพิ่มและลดลงประมาณ ±20% แล้วคำนวณผลลัพธ์ใหม่ เพื่อเปรียบเทียบกับโมเดลเดิมว่ามีการเปลี่ยนแปลงมากน้อยเพียงใด

**2. พื้นที่ Robust และไม่แน่นอน**

ผลการวิเคราะห์พบว่าพื้นที่ที่มีค่าความเสี่ยงคงที่แม้มีการเปลี่ยนน้ำหนักถือเป็นพื้นที่ที่มีความเสถียร (Robust) ในขณะที่พื้นที่ที่มีการเปลี่ยนแปลงของค่าความเสี่ยงอย่างชัดเจนถูกจัดเป็นพื้นที่ที่มีความไม่แน่นอน ซึ่งมักอยู่บริเวณขอบเขตของพื้นที่เสี่ยง

**3. ผลต่อความเชื่อมั่นของโมเดล**

Sensitivity Analysis แสดงให้เห็นว่าโมเดลมีความเสถียรในภาพรวม แต่ยังมีบางพื้นที่ที่ไวต่อการเปลี่ยนแปลงของน้ำหนัก ซึ่งสะท้อนว่าความเชื่อมั่นของโมเดลขึ้นอยู่กับความแม่นยำของการกำหนดน้ำหนัก และควรมีการตรวจสอบเพิ่มเติมในพื้นที่ที่มีความไม่แน่นอนสูง

 # **ภารกิจที่ 4: Validation กับข้อมูลจริง**

**1. การเปรียบเทียบกับ JRC Global Surface Water**

โมเดลความเสี่ยงน้ำท่วมถูกตรวจสอบโดยเปรียบเทียบกับข้อมูล JRC Global Surface Water (Occurrence) ซึ่งใช้เป็นตัวแทนของพื้นที่ที่มีน้ำปรากฏในระยะยาว โดยนำมาคำนวณร่วมกับผลลัพธ์ของโมเดลเพื่อสร้าง confusion matrix และประเมินค่า Accuracy

**2. (กรณีโมเดลอื่น เช่น พืช / ไฟป่า — เชิงแนวคิด)**

ในกรณีของโมเดลประเภทอื่น เช่น ความเหมาะสมของพืชหรือไฟป่า สามารถใช้ข้อมูล MODIS Land Cover หรือ Burned Area ในการตรวจสอบความถูกต้อง โดยใช้แนวทางการเปรียบเทียบเชิงพื้นที่ในลักษณะเดียวกัน

**3. ข้อจำกัดของ Validation และการใช้งานจริง**

เนื่องจากข้อมูลน้ำท่วมจริงในระดับพื้นที่มีข้อจำกัดด้านความพร้อมและความละเอียด จึงจำเป็นต้องใช้ข้อมูล proxy เช่น JRC แทน อย่างไรก็ตาม ในการใช้งานจริงควรมีการเก็บข้อมูลภาคสนามหรือใช้ข้อมูลเหตุการณ์น้ำท่วมจริง (Ground Truth) เพื่อเพิ่มความแม่นยำในการตรวจสอบโมเดล

# **อธิบาย methodology**


**1. ถ้ามี Ground Truth จริง Accuracy จะอยู่ระดับไหน? เพราะอะไร**

หากมีข้อมูล Ground Truth จริง คาดว่า Accuracy ของโมเดลจะอยู่ในระดับปานกลางถึงค่อนข้างดี (ประมาณ 70–85%) เนื่องจากโมเดลใช้ข้อมูลเชิงกายภาพหลักที่เกี่ยวข้องกับน้ำท่วมจริง เช่น ความสูง ปริมาณฝน และระยะห่างจากแหล่งน้ำ อย่างไรก็ตาม การใช้ข้อมูล proxy อย่าง JRC แทนเหตุการณ์น้ำท่วมจริงอาจทำให้ผลลัพธ์ดูดีกว่าความเป็นจริง และเมื่อใช้ข้อมูลภาคสนามจริงที่มีความซับซ้อนมากขึ้น Accuracy อาจลดลงเล็กน้อย

**2. ปัจจัยที่ไม่ได้ใส่ในโมเดลแต่สำคัญคืออะไร? ทำไมไม่ใส่**

ปัจจัยที่สำคัญแต่ไม่ได้รวมในโมเดล ได้แก่ ระบบระบายน้ำ (drainage system), การใช้ที่ดินเชิงรายละเอียด (เช่น impervious surface), ความหนาแน่นประชากร และข้อมูลน้ำฝนรายชั่วโมง ซึ่งมีผลโดยตรงต่อการเกิดน้ำท่วม แต่ไม่ได้ถูกนำมาใช้เนื่องจากข้อจำกัดด้านความพร้อมของข้อมูล ความละเอียดของข้อมูล และความซับซ้อนในการประมวลผลในระดับการศึกษา

**3. โมเดลนี้เหมาะกับ scale ไหน เพราะอะไร**

โมเดลนี้เหมาะสมกับการใช้งานในระดับจังหวัดมากที่สุด เนื่องจากใช้ข้อมูลเชิงพื้นที่ความละเอียดปานกลาง (เช่น Sentinel-2 และ DEM) และมีการ generalize ปัจจัยต่าง ๆ ซึ่งเพียงพอสำหรับการวิเคราะห์ภาพรวม แต่ยังไม่ละเอียดพอสำหรับการใช้งานในระดับตำบลหรือพื้นที่ย่อยที่ต้องการข้อมูลเชิงลึกมากขึ้น

**4. ถ้าต้องอัปเดตโมเดลทุกปี ต้องทำอะไรซ้ำ / อะไรคงที่**

หากต้องอัปเดตโมเดลทุกปี ขั้นตอนที่ต้องทำซ้ำ ได้แก่ การดาวน์โหลดข้อมูลใหม่ (เช่น rainfall, satellite imagery), การ preprocess ข้อมูล และการคำนวณค่า index ต่าง ๆ ขณะที่ขั้นตอนที่คงที่ ได้แก่ โครงสร้างของโมเดล วิธีการ normalize การกำหนดน้ำหนัก และสมการในการรวมปัจจัย ซึ่งสามารถใช้ซ้ำได้โดยไม่ต้องเปลี่ยนแปลง

# **อธิบาย**
**1. วิธีที่เลือกใช้**
โครงการนี้ใช้วิธี Multi-Criteria Analysis (MCA) ร่วมกับ Weighted Linear Combination (WLC) ในการสร้างแบบจำลองความเสี่ยงน้ำท่วม โดยนำข้อมูลเชิงพื้นที่จาก Google Earth Engine (GEE) เช่น Elevation, Slope, Rainfall, Distance to Water และ Land Cover มาประมวลผล จากนั้นทำการ Normalize ปัจจัยให้อยู่ในช่วง 0–1 และรวมเข้าด้วยกันตามค่าน้ำหนักที่กำหนด เพื่อสร้าง Flood Risk Index และแสดงผลในรูปแบบแผนที่เชิงพื้นที่ นอกจากนี้ยังมีการทำ Sensitivity Analysis และ Validation โดยใช้ข้อมูล JRC Global Surface Water เพื่อประเมินความถูกต้องของโมเดล

---

**2. เหตุผลที่เลือกวิธีนี้**
เลือกใช้วิธี MCA เนื่องจากเป็นวิธีที่เหมาะสมสำหรับการวิเคราะห์ปัญหาที่มีหลายปัจจัยร่วมกัน และสามารถอธิบายความสัมพันธ์ของปัจจัยเชิงพื้นที่ได้อย่างชัดเจน อีกทั้งยังสามารถนำไปประยุกต์ใช้ในระบบ GIS ได้ง่าย และไม่ต้องการข้อมูลจำนวนมากหรือซับซ้อนเหมือนวิธี Machine Learning :contentReference[oaicite:0]{index=0} นอกจากนี้ MCA ยังเหมาะกับการวิเคราะห์ในระดับพื้นที่กว้าง เช่น ระดับจังหวัด

---

**3.ข้อจำกัดที่พบ**
โมเดลนี้มีข้อจำกัดหลักคือการใช้ข้อมูล proxy เช่น JRC Global Surface Water แทนข้อมูลน้ำท่วมจริง (Ground Truth) ซึ่งอาจไม่สะท้อนเหตุการณ์น้ำท่วมในช่วงเวลาสั้นได้อย่างแม่นยำ นอกจากนี้ยังไม่ได้รวมปัจจัยสำคัญบางอย่าง เช่น ระบบระบายน้ำ ความหนาแน่นของสิ่งปลูกสร้าง หรือข้อมูลฝนรายชั่วโมง รวมถึงการกำหนดน้ำหนักแบบ Expert Knowledge อาจมีความเป็นอัตวิสัย และส่งผลต่อผลลัพธ์ของโมเดล

**4.สิ่งที่ค้นพบ**
ผลการวิเคราะห์พบว่าโมเดลสามารถจำแนกพื้นที่ไม่ท่วมได้ดีในภาพรวม และสามารถระบุพื้นที่เสี่ยงน้ำท่วมได้ในบางบริเวณที่สอดคล้องกับข้อมูลจริง อย่างไรก็ตามยังพบว่ามีการประเมินน้ำท่วมเกินจริงในบางพื้นที่ (False Positive) และพลาดพื้นที่น้ำท่วมบางส่วน (False Negative) ซึ่งสะท้อนให้เห็นถึงความไม่แน่นอนของโมเดลในบางบริเวณ โดย Sensitivity Analysis แสดงให้เห็นว่าโมเดลมีความเสถียรในระดับหนึ่ง แต่ยังไวต่อการเปลี่ยนแปลงของน้ำหนักในบางพื้นที่
